Representation of Class and Object in UML diagram:¶

Links – We use a link to represent a relationship between two objects¶

Notation of link:¶

Association:¶

If two classes in a model need to communicate with each other, there must be a link between them, and that can be represented by an association (connector).

Association is a reference relationship between two objects (or classes).

Notation for association :¶

Whenever an object uses another it is called an association.We use association when one object references members of the other object. Association can be uni-directional or bi-directional. We use an arrow to represent association.

Types of Association:¶

1.Uni-directional – The object of Order class is associated with an object of Customer class.¶

class Customer:
    def __init__(self,name,phone):
        self.name=name
        self.phone=phone
class Order:
    def __init__(self,OI,OT,customer):
        self.order_id=OI
        self.order_type=OT
        self.customer=customer
    def display(self):
        print(self.customer.name)
        print(self.customer.phone)
        print(self.order_id)
        print(self.order_type)

3.Multidirectional – multiplicity of an association by adding multiplicity adornments to the line denoting the association. The example indicates that a Student has one or more Instructors:¶

class Instructor:
    def __init__(self,name,subject):
        self.ins_name=name
        self.subject=subject
class Student:
    def __init__(self,name,roll,branch,instructor1,instructor2):
        self.stu_name=name
        self.stu_roll=roll
        self.stu_branch=branch
        self.instructor1=instructor1
        self.instructor2=instructor2
    def display(self):
        print(self.instructor1.ins_name)
        print(self.instructor1.subject)
        print(self.instructor2.ins_name)
        print(self.instructor2.subject)

Iobj1=Instructor("amir","IE")
Iobj2=Instructor("nitin","SOM")
Sobj1=Student("Rahul","12345","Mechanical",Iobj1,Iobj2)
Sobj1.display()
