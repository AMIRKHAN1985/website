Inheritance¶

Inheritance is the capability of one class to derive or inherit the properties from some another class. The benefits of inheritance are:

  1. It represents real-world relationships well.

  2. It provides reusability of a code. We dont have to write the same code again and again. Also, it allows us to add more features to a class without modifying it.

  3. It is transitive in nature, which means that if class B inherits from another class A, then all the subclasses of B would automatically inherit from class A.

2. Multiple inheritance:¶

When a child class inherits from multiple parent classes, it is called as multiple inheritance.

In [25]:
class A:
    def __init__(self,name1):
        self.name1=name1
class B:
    def __init__(self,name2):
        self.name2=name2
class C(A,B):
    def __init__(self,name1,name2):
        A.__init__(self,name1)
        B.__init__(self,name2)
    def display(self):
        print(self.name1,self.name2)
obj=C("jack","rahul")
obj.display()

super function¶

The super() function is used to give access to methods and properties of a parent or sibling class.

The super() function returns an object that represents the parent class.

In [42]:
class C:
    name="i am in class A variable"
    def __init__(self):
        self.name="i am in class A constructor"
       
class B(C):
    name="i am in class B variable"
    def __init__(self):
        super().__init__()

Advantage of using super()¶

In [18]:
class Mammal:
  def __init__(self, mammalName):
    print(mammalName, 'is a warm-blooded animal.')
    
class Dog(Mammal):
  def __init__(self,name):
    print('Dog has four legs.')
    super().__init__(name)
    
d1 = Dog("Dog")

Dog has four legs.
Dog is a warm-blooded animal.

Since we do not need to specify the name of the base class when we call its members, we can easily change the base class name (if we need to).
